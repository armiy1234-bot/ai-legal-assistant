## Приоритет задач (выполнять по порядку)

### Фаза 1: Стабильность и баги
- Пофиксить скролл на мобилках
- Починить compact-режим AiAssistant
- Исправить rate-limit in-memory fallback
- Обработка ошибок Mistral (таймауты, падения)
- Проверить next-auth сессию на первом заходе

### Фаза 2: Функционал
- i18n (русский/английский) через React Context + localStorage
- Система тем (2-4): emerald, ocean, royal + dark варианты
- Улучшить system prompt: ссылки на ГК РФ, УК РФ, КоАП, Конституцию
- Категоризация вопросов (гражданские, семейные, трудовые, жилищные, ДТП и т.д.)
- Страница логина (упрощённая, сохранить VK ID)

### Фаза 3: Маркетинг и продвижение
- Добавить блок отзывов на лендинг
- Форма обратной связи на видном месте
- FAQ-секция с частыми юридическими вопросами
- SEO-оптимизация: meta-теги, Open Graph, заголовки, структура
- Блог/статьи по популярным юридическим темам (через web_search + easy-search)

### Фаза 4: Мониторинг (регулярная задача)
Каждую неделю запускать:
1. SEO-аудит через `seo-optimizer` и `technical-seo-checker`
2. Проверка скорости и Core Web Vitals
3. Анализ конкурентов через web_search (что нового у Правовед, 9111 и др.)
4. Отчёт в `memory/monitoring/YYYY-MM-DD-seo.md`
5. При падении трафика или ошибках — создать GitHub Issue и уведомить

### Фаза 5: Развитие
Изучи конкурентов (`АНАЛИЗ_КОНКУРЕНТОВ.md`) и добавь:
- Раздел "Для юристов" (премиум доступ к аналитике)
- Экспорт ответа в PDF
- Гайды/чек-листы по статьям законов
- Счётчик бесплатных запросов с уведомлением
