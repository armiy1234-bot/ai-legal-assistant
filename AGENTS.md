## Текущая архитектура
- `app/layout.tsx` — корневой layout, AiAssistant сайдбар слева, десктоп/мобилка
- `app/page.tsx` — лендинг (hero, stats, features, categories, CTA, footer)
- `app/globals.css` — CSS переменные (сейчас только :root и .dark, emerald тема)
- `app/api/chat/route.ts` — POST handler для AI чата
- `app/api/auth/[...nextauth]/route.ts` — NextAuth handlers
- `lib/auth.ts` — next-auth lazy config (VK, Telegram, Phone провайдеры)
- `lib/db/index.ts` — getDb() lazy drizzle инициализация
- `lib/db/schema.ts` — таблицы: users, legal_queries, court_cases
- `lib/rate-limit.ts` — in-memory fallback rate limit (без Upstash Redis)
- `lib/ai/mistral-client.ts` — Mistral AI клиент через @ai-sdk/openai
- `components/sidebar/ai-assistant.tsx` — Apple-стиль чат
- `components/chat/` — компоненты чата
- `tailwind.config.ts` — emerald тема, Inter шрифт, apple shadows

## Миссия
Сделать качественный бесплатный AI-помощник по юридическим вопросам для России и СНГ, который будет полезен обычным людям (не юристам). 

Конкуренты для анализа (посмотри их функционал через web_search):
- **ЧатGPT / DeepSeek** — просто AI, нет специализации
- **КонсультантПлюс / Гарант** — справочно-правовые системы, но сложные для людей
- **СберПраво / норм.рф** — онлайн юридические консультации
- **LegalTech AI** — зарубежные аналоги (DoNotPay, Lawgeex)
- **Тинькофф Юрист** — базовые шаблоны документов

## Технические требования
- После каждого изменения запускать `npm run build` — сборка не должна падать
- Не удалять существующие env переменные
- Новые файлы создавай в соответствующих директориях (lib/, components/, app/, etc.)
- Сохранять стиль: Inter, скругления, мягкие тени, чистый минимализм
- Проект на TypeScript — всё должно быть типизировано
- Коммитить после каждой задачи: `git add -A && git commit -m "feat: ..." && git push`
- Если что-то не уверен — используй `web_search` для поиска ответов
- Если задача сложная — разбей на подзадачи и делай пошагово
